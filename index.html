Conceitos Git e GitHub

O que é Git?
Git é um sistema de controle de versão distribuído, criado para gerenciar e acompanhar alterações em arquivos de projetos, especialmente de código-fonte. Ele permite que você registre o histórico de modificações, trabalhe em diferentes versões e colabore com outras pessoas de forma eficiente.

O que é GitHub?
GitHub é uma plataforma online que hospeda repositórios Git, facilitando o compartilhamento de projetos, colaboração em equipe e integração com ferramentas de desenvolvimento. No GitHub, você pode contribuir com projetos open source, gerenciar issues e realizar revisões de código.

  • Repository: Pasta do projeto com controle de versão.
  • Commit: Registro de uma alteração feita no projeto, acompanhado de uma mensagem descritiva.
  • Branch: Linha paralela de desenvolvimento, útil para trabalhar em novas funcionalidades sem afetar o código principal.
  • Merge: Combina alterações de diferentes branches.
  • Push: Envia commits locais para o repositório remoto (GitHub).
  • Pull: Baixa alterações do repositório remoto para o local.
  • Clone: Faz uma cópia completa de um repositório remoto.
  • Untracked: Arquivo novo que o Git ainda não monitora.
  • Modified: Arquivo alterado desde o último commit.
  • Staged: Arquivo pronto para ser incluído no próximo commit.
  • Committed: Arquivo salvo no histórico do Git.

Fluxo de Trabalho com Git e GitHub

O fluxo de trabalho com Git e GitHub segue uma sequência lógica para garantir organização, colaboração e segurança no desenvolvimento de projetos. Veja o passo a passo:

  1. Clonar o repositório:
    Baixe uma cópia do projeto para sua máquina.
    git clone <url-do-repositorio>
  2. Criar uma branch:
    Crie uma linha de desenvolvimento separada para trabalhar em novas funcionalidades ou correções sem afetar o código principal.
    git checkout -b nome-da-branch
  3. Fazer alterações:
    Edite, adicione ou remova arquivos conforme necessário. Teste suas mudanças localmente.
  4. Adicionar arquivos ao stage:
    Prepare os arquivos modificados para serem incluídos no próximo commit.
    git add . ou git add nome-do-arquivo
  5. Realizar um commit:
    Salve um ponto do histórico do projeto com uma mensagem clara sobre o que foi alterado.
    git commit -m "Descreva sua alteração"
  6. Sincronizar com o repositório remoto:
    Envie suas alterações para o GitHub.
    git push origin nome-da-branch
  7. Abrir um Pull Request:
    No GitHub, solicite a revisão e integração das suas mudanças na branch principal. Explique o que foi feito e aguarde aprovação.
  8. Atualizar seu repositório local:
    Antes de começar novas alterações, baixe as últimas atualizações do projeto.
    git pull origin main
  9. Mesclar branches:
    Após aprovação do Pull Request, as alterações são integradas à branch principal. Se necessário, faça o merge localmente:
    git checkout main
    git merge nome-da-branch

Dicas:
- Sempre trabalhe em branches separadas para cada tarefa.
- Escreva mensagens de commit claras e objetivas.
- Sincronize seu repositório local com frequência usando git pull.
- Revise e teste suas alterações antes de enviar para o repositório remoto.

Como Resolver Conflitos

Conflitos acontecem quando duas pessoas alteram a mesma linha de um arquivo ou arquivos diferentes que afetam o mesmo trecho do código. O Git não consegue decidir qual alteração manter e pede que o usuário resolva manualmente.

  1. Ao tentar fazer um merge ou pull, o Git avisa sobre o conflito.
  2. Abra o arquivo indicado e procure por marcações como <<<<<<< HEAD e >>>>>>> branch.
  3. Escolha qual versão manter ou combine as alterações.
  4. Remova as marcações do Git.
  5. Adicione o arquivo resolvido ao stage: git add arquivo
  6. Finalize com um commit: git commit -m "Resolve conflito"

Dica: Use editores como VS Code, que facilitam a visualização e resolução de conflitos.
